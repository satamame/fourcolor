# 四色問題プロジェクト

四色問題を「地図のヘックス（ハニカム）化」と「same_value エッジ表現」という独自の切り口から
アルゴリズム的に攻めるプロジェクト。

## 方針（要約）

- ヘックス化は問題を「易しくする」のではなく「正規化する」変換。
  座標・走査順・データ構造が手に入るので、その土俵で実験して構造を探す。
- 「衝突したときの修正」は古典の **Kempe 鎖** に対応する。
  貪欲彩色＋Kempe 修復をヘックス盤上で観察するのが当面の中心課題（Phase 2）。
- 出力の正しさは常に独立した検証器でチェックし、
  遅いが確実なバックトラック彩色をオラクル（答え合わせ）として持つ。

## 使い方

Python 3.13、標準ライブラリのみ。プロジェクトルートで:

```
# テスト
python -m unittest discover -s tests -v

# サンプル: 地図のハニカムグラフ表現 (検証 + SVG 3面図の出力)
python samples/sample01_k4.py      # 4カ国・4色が必要な最小の例
python samples/sample02_seven.py   # 7カ国・3色で足りる少し大きい例
python samples/sample03_point.py   # 7カ国が1点に集まる例 (無色ノード)

# デモ: ランダム地図の生成 → 4彩色 → 盤面表示 (旧モデル: ノード=六角形セル)
python demo.py 10 0    # 盤サイズ m=10, seed=0
```
