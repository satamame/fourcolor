# アイデア: ラベルの4値化（行操作と斜め重ね合わせ）

**状態: おぼろげ。** 実装前の検討用に、アイデアの原型と評価を書き留めておく。
[docs/05](05_アイデア_行ラベリング.md)/[06](06_行ラベリング結果.md) の行ラベリングが土台。

## 0. 共通の前提

「ラベルを4種類まで減らす」は「4彩色を構成する」ことそのもの。
したがって「必ず成功する単純な操作列」が見つかるならそれは四色定理の新証明であり、
docs/01 §2.3 の落とし穴チェックリストに照らすと、そう簡単にはいかないはず。

期待値は「簡単に減らせる方法の発見」ではなく、
**操作をどう設計すると、どこで・なぜ詰まるかが見えるようになるか**に置く。
その観点で両アイデアとも観察装置として価値がある。

## 1. アイデア1: 縦エッジを考慮した行ラベルの操作による4値化

### 原型

行ラベル (r, c) を、縦エッジの値（same_value）を考慮しながら操作していくことで、
ラベルの種類を4つまで減らせないか。

### 評価

これは実質、ロードマップの「(r, c) 順の貪欲彩色」を**ラベル書き換え操作**として
見たもの。方向性は Phase 2 の計画と一致している。ただし原理的な障害が2つある。

1. **「あとで判明する等値制約」問題。**
   下に開いた U 字型の国は行 r に2つのセグメントとして現れ、
   同じ国だと分かるのは下の行を処理したとき。
   上から一方向に処理する限り、ラベル確定後に
   「実は同じ値にすべきだった」が必ず起こりえる。
2. **部分彩色は常には延長できない。**
   等値情報をすべて知っていても、上の行までの4値割り当てが
   次の行に延長できる保証はない（貪欲彩色が詰まる現象と同じ）。
   「一部の頂点の色が決まった平面グラフの彩色を完成できるか」
   （precoloring extension）は平面グラフでも NP 完全という既知の結果があり、
   **修正なしの一方向掃引には理論的な壁がある**。

→ 活かす形: **過去の行のラベルを書き換える修正操作（Kempe 鎖に相当）を
操作系に含める**設計。制約を保存する書き換え規則を列挙し、
どの規則がどこで必要になるかを実験で測る。

## 2. アイデア2: 斜め方向ラベリングとの重ね合わせによる4値化

### 原型

同じハニカムグラフに対し、行ではなく**斜め方向**の一連のノード単位で
ラベル付けしたものを作る。2つ（以上）のラベリングを重ね合わせることで、
ラベルを4種類まで減らせないか。

### 評価

筋が良い。理由は2つ。

1. **盤の構造と合っている。**
   三角形盤には行（水平）のほかに左斜め・右斜めの2方向の「行」があり、
   3方向はハニカム格子の3種類のエッジ方向と対応する。
   盤は120°回転対称なので、斜めラベリングは
   **座標変換して rowlabel.py を再利用するだけ**で実装できる（安い）。
2. **古典理論と同型。**
   4色を2ビットの組 (a, b)（a, b ∈ {0, 1}）と見ると、
   4彩色は「2つの2値ラベリングの重ね合わせ」に分解される。
   「国境を2クラスに分け、各クラスだけ見ると2彩色可能になるようにできるか」
   という形の定式化は、**Tait の3辺彩色**（docs/01 §2.1。四色定理と同値）と
   本質的に同じもの。つまりこのアイデアは、既知の理論が
   「ここが核心だ」と指している場所に独自ルートで到達している。

即座に見える障害:

- 行ラベルの c の偶奇を2値ラベルとして使おうとすると、
  **c はセグメント単位なので同じ国のセグメント間で偶奇が揃わない**
  （例: サンプル02 の D は行5で c=2 と c=4）。
  重ね合わせの前に各方向のラベルを国単位で整合させる操作が必要で、
  そこにアイデア1と同じ難しさが戻ってくる。
  難しさが消えるのではなく
  「2つの2値問題の整合」という形に**変形される**——
  ただし、変形された形の方が観察しやすい可能性は十分ある。

## 3. 検討の結論（2026-06-11）

その後の検討で次が整理された。

- **2レイヤーに分ける**:
  (i) **同一性レイヤー** = Union-Find が持つ国の分割（方向に依存しない）。
  (ii) **座標レイヤー** = 方向ごとのセグメント番号 c（rowlabel.py の出力そのもの。
  「素の c」。c mod 2 は「その線でそこまでに横切った国境数の偶奇」）。
  アイデア1の正規化は (i) に向かう操作で方向依存性を消す。
  アイデア2は (ii) の方向依存性を使う。**2つは逆を向いているので、レイヤーを
  混ぜない**（リネームは座標レイヤーを壊すので案2では行わない。
  国単位の整合チェックは Union-Find 側で行う）。
- **アイデア1は「併合計算」として定式化できる**:
  強制併合（同じ国と判明したラベルの統一。常に安全・合流的。
  遠回りで判明した同一性も遡及リネームで矛盾なく反映できる）と、
  選択併合（隣接しない国の同色化。間違いうる）の2種類の操作で、
  4彩色の構成 = 選択併合の列（Zykov の縮約と同型）。
  詰まり = 縮約後のグラフへの K5 の出現として機械検出できる。
- **アイデア2の検証すべき仮説の形**:
  「行方向の走査で横切る国境は常に同じビットを反転させるように、
  ビット割り当てを選べるか」。→ 実験2A
  ([docs/08](08_実験2A_方向とビット分解.md)) で検証した。
  結果: 国なしの盤では完全整合の彩色が構成できるが、国を作ると
  最良でも 0.5〜0.8 に落ちる。素朴な案2B は不成立、ただし構造は半分以上残る。

## 4. 具体化に向けた論点（未決）

実装の前に決めたいこと。

**アイデア1:**

- 許す書き換え操作の語彙は何か
  （国単位の値の入れ替え／2値だけの連結成分内での交換 = Kempe 交換／行内限定の操作／…）
- 行の処理で何を確定するか（セグメントの値か、国の値か。国の初出時に確定 = 貪欲彩色）
- 「詰まり」の定義と、修正の波及範囲（何行さかのぼったか）の計測方法

**アイデア2:**

- 斜め方向の座標変換（120°回転）の具体式
- 2値化の方法（c mod 2 をそのまま使う案は国単位で破綻することが分かっている。
  ではどう整合させるか）
- 逆向きの実験から入る案: baseline の4彩色を2ビットに分解し、
  「a が変わる国境／b が変わる国境／両方変わる国境」の3クラスが
  盤面の3方向・3方向のラベリングとどう対応するかを観察する
  （= 重ね合わせの「あるべき姿」を先に見る）
