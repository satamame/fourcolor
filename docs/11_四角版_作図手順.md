# 四角版 作図手順書（地図 → ①横ラベルグラフ / ②縦ラベルグラフ / ③2軸ラベル表）

四角分割した地図データを人が作り、そこから ① 横ラベル付きグラフ、
② 縦ラベル付きグラフ、③ 2軸ラベル表 を生成・可視化するための手順。
ブランチ `square-grid-prototype` の四角版プロトタイプを使う。

## 0. 前提

- Python 3.13、標準ライブラリのみ。リポジトリのルートで作業する。
- 作例: [samples/square_mymap.py](../samples/square_mymap.py)（このテンプレを複製して使う）。

## 1. 地図データを作る

### 座標と分割

- 盤は `ROWS × COLS` の正方形グリッド。マスの座標は `(r, c)`。
  - `r` = 行（**上から 0, 1, 2, …**）
  - `c` = 列（**左から 0, 1, 2, …**）
- 隣接は上下左右（辺を共有）だけ。**斜め（角だけ）は隣接ではない。**

### 国の塗り方のルール

1. すべてのマスに国名（文字列）を割り当てる。
2. **同じ国のマスは辺でつながっている**こと（連結。飛び地は不可）。
3. **5カ国以上が1点（格子点）に集まる場所だけ**、そのマスを `None` にする
   （= 無色ノード。格子点には4マスしか集まれないため）。4カ国までは無色ノード不要。

### 書き方（テンプレを複製）

`samples/square_mymap.py` を複製し、`ROWS`, `COLS`, `COUNTRY_OF` を書き換える。
最低限必要なのは次の3つ:

```python
from fourcolor.mapcheck import build_edges
from fourcolor.square import SquareGrid

ROWS, COLS = 3, 4

COUNTRY_OF = {
    (0, 0): "A", (0, 1): "A", (0, 2): "B", (0, 3): "B",
    (1, 0): "A", (1, 1): "C", (1, 2): "C", (1, 3): "B",
    (2, 0): "D", (2, 1): "D", (2, 2): "C", (2, 3): "B",
}

def build_sample():                      # この関数名は固定（生成器が呼ぶ）
    grid = SquareGrid(ROWS, COLS)
    return grid, COUNTRY_OF, build_edges(grid, COUNTRY_OF)
```

- `COUNTRY_OF` は **全マス**を埋める（`ROWS×COLS` 個）。埋め忘れるとエラー。
- 無色ノードを置くときはその座標を `(r, c): None` にする。

## 2. 図を生成する

リポジトリのルートで、モジュール名（拡張子なし）を渡して実行:

```
python samples/square_figures.py square_mymap
```

次の2ファイルが `samples/` に出力される（SVG。ブラウザや画像ビューアで開く）:

- `samples/square_mymap_labels.svg` … ① 横ラベルグラフ と ② 縦ラベルグラフ
- `samples/square_mymap_table.svg`  … ③ 2軸ラベル表

（引数なしで実行すると、同梱のサンプル01/02 の図を再生成する。）

## 3. 出力物の見方

### ①②　`_labels.svg`（横ラベル / 縦ラベルのグラフ）

左右に同じ地図が2枚並ぶ。

- 左 = **横ラベル h**（各行を左→右にスキャンして採番）
- 右 = **縦ラベル v**（各列を上→下にスキャン）
- 各マス: 塗り = 国の色、中央の数字 = そのマスのラベル値。
- **太線** = 国境（隣り合う別の国の境）。
- **白い破線のマス** = 無色ノード（ラベルは持つが色は持たない）。

### ③　`_table.svg`（2軸ラベル表）

- 上のヘッダ = 横ラベル h、左の見出し = 縦ラベル v。
- セル: 国の色で塗られ、数字は**仮色値**（初期は国ごとに一意）。
- **白い破線セル** = 無色ノード（値 0）。
- **空白の列・行** = 区切り（ブロックの境界。ラベルが飛んだ位置）。
- 読み方の要点: 同じ色が同じ行・列に並ぶのが「同ラベル＝同色」、
  隣り合うブロックの色が違うのが「差1＝別色」。各ブロックにマスは1個。

## 4. （任意）正しさを確認する

数値での検算をしたい場合、ルートで:

```
python -c "import sys; sys.path[:0]=['.','samples']; \
import square_mymap as m; from fourcolor.squaretable import \
build_square_table, table_equivalence, is_grid_realizable; \
from fourcolor.baseline import solve; \
g,co,e=m.build_sample(); t,_,_=build_square_table(g,e,co); \
print('国→仮色値', t.cid); \
print('同値性(等式=国/差1=国境/ブロック最大)', table_equivalence(g,e,co)); \
print('格子に再構成可', is_grid_realizable(t)); \
print('3色で塗れる?', solve(e,3) is not None)"
```

- `table_equivalence` の1つ目が `True` … 表が地図と同値（**`False` なら国が
  非連結など、地図データの不備**）。
- `is_grid_realizable` が `(True, None)` … 健全（格子＝平面地図として整合）。
- `solve(e, k)` … k 色で塗れるかの判定（`None` なら不可）。何色必要かを試せる。

## 5. よくある間違い

- **国が非連結**（同じ名前のマスが離れている）→ `table_equivalence` の1つ目が
  `False` になる。連結になるよう塗り直す。
- **マスの埋め忘れ** → `build_edges` でエラー。全 `ROWS×COLS` マスを埋める。
- **5カ国以上が1点に集まる**のに無色ノードを置いていない → その点を表現できない。
  集まる点のマスを `None` にする。
- **斜め隣接を国境のつもりにしている** → 四角格子では角だけの接触は隣接でない。
  隣接させたいなら辺を共有させる。
