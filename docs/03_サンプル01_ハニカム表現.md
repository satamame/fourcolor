# サンプル01: 簡単な地図とそのハニカムグラフ表現

仕切り直しの出発点。
アイデア1「地図をハニカム構造のグラフに変換する」を最小の具体例にしたもの。

## モデルの定義

- **ノード = ハニカムグラフの頂点**。これにより「各ノードの隣接数は最大3」が文字どおり成り立つ。
- 幾何的な対応：ハニカム格子の頂点は、三角格子の**小三角形**と1対1に対応する（双対関係）。
  つまり「地図を小三角形に分割し、1つの国＝連結な小三角形の集合」と考えるのと同じこと。
  - ▲（上向き三角形）と ▽（下向き三角形）が交互に並び、各三角形は辺を共有する最大3個の三角形とだけ隣接する。
  - この三角形の隣接グラフを点と線で描くと、ちょうど蜂の巣（六角形）模様になる。
- **エッジ属性 same_value**（アイデア2）：
  - `True` … 同じ国の内部（同色強制）
  - `False` … 国境（異色強制）。塗り分けの制約はこちらだけに効く。

実装は [fourcolor/honeycomb.py](../fourcolor/honeycomb.py)。
盤面は大きな正三角形を1辺 size 個に分割したもので、ノード座標は (r, p)
（r=行 1..size、p=行内位置 1..2r-1、p奇数=▲ / p偶数=▽）。

## サンプルの地図

中央の小さな国 D を、3つの国 A（下）、B（左上）、C（右）が取り囲む。
4つの国は互いにすべて隣接する（国の隣接グラフ = K4）ので、**4色が必要な最小の例**。

1辺4の盤面（ノード16個）への割り当て：

```
            B
          B B C          B = 7ノード (左上)
        B B D C C        C = 5ノード (右)
      B B A A A C C      D = 1ノード (中央)、A = 3ノード (下)
```

検証済みの性質（[samples/sample01_k4.py](../samples/sample01_k4.py) の `check_sample` が機械チェック）：

| 項目 | 値 |
|---|---|
| ノード数 / エッジ数 | 16 / 18 |
| 内部エッジ (same_value=True) | 12 |
| 国境エッジ (same_value=False) | 6 — 国の組6通りそれぞれにちょうど1本 |
| 最大隣接数 | 3（ハニカムグラフの性質） |
| 彩色 | 4色で可、3色では不可（K4 なので） |

## 再生成

```
python samples/sample01_k4.py
```

検証結果を表示し、3面図（地図 / 三角形分割 / ハニカムグラフ）の
[samples/sample01_k4.svg](../samples/sample01_k4.svg) を出力する。

## このサンプルが固定する「入力形式」

塗られていない地図 = `edges = {(u, v): {"same_value": bool}}` だけが与えられた状態。
彩色アルゴリズムはこの形式を入力とし、出力は `{ノード: 色}`。
正しさは常に `fourcolor.verify.violations()` で判定する。

## メモ

- 初代の実装 [fourcolor/hexgrid.py](../fourcolor/hexgrid.py)（TriBoard）は「ノード=六角形セル、隣接最大6」
  の初期モデル。今後の標準は本サンプルの honeycomb モデル（隣接最大3）とする。
  TriBoard ベースの mapgen / demo はランダム地図の供給源としては引き続き使える。
- 次のステップ：このサンプルを入力に「インデックス順の貪欲彩色＋衝突時の修正」を動かしてみる。
  もっと大きなサンプル（修正が実際に必要になる地図）も honeycomb モデルで作る。
