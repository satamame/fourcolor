# 四角版プロトタイプ（格子グラフ）

**状態: プロトタイプ。** ブランチ `square-grid-prototype` で試作中。
三角分割（ハニカム）の代わりに**四角分割（格子グラフ）**で 2軸ラベル表を作り、
[docs/09](09_アイデア_2軸ラベル表.md) の議論が成り立つかを確認する。

- 格子モデル: [fourcolor/square.py](../fourcolor/square.py)（`SquareGrid`、次数≤4）
- 2軸表: [fourcolor/squaretable.py](../fourcolor/squaretable.py)
- サンプル: `python samples/square_sample01.py`（3×3 の K4）、
  `python samples/square_sample02_point.py`（5カ国が1点・無色ノード）
- テスト: [tests/test_square.py](../tests/test_square.py)

## 三角版と共有するもの

セルの値の意味（`≥1` 有色 / `0` 無色 / `-1` 空き / `-2` 区切り）、採番規則
（True→同・False/None→+1、グループ間 +2）、`AxisTable` クラスはそのまま再利用。
違いは**スキャン順だけ**:

- 横ラベル: 各行を左→右
- 縦ラベル: 各列を上→下（三角の**ジグザグが不要**）

## すっきりする点（確認済み）

1. 縦スキャンのジグザグが消える。
2. 辺の方向が横・縦の2つだけ。横ラベル＝横の辺、縦ラベル＝縦の辺と1対1。
3. **各ブロックが最大1セル**。四角では「列ブロック＝格子の行」「行ブロック＝
   格子の列」で交点はマス1個。三角の ▲▽ ペア（最大2セル・「／」向き）が消え、
   ただの格子配置になる。

## 検証結果（test_square.py、3地図）

K4（3×3）・縦縞・4×4ブロックの地図で、いずれも:

- 等式（同じ横/縦ラベル）の推移閉包 = 国の分割 ✓
- 差1（横/縦ラベル）の異色ペア = baseline の国境 ✓
- ブロック内の最大セル数 = 1 ✓

→ 中心の議論（2軸表が地図と同値、色を減らす作業場になる）は四角でも**破綻しない**。

## 健全性の検討（四角版）

### 健全性の定義（循環を避ける）

最終目標が「格子由来の表は4色で塗れる」の**証明**なら、健全性の判定に
「4色で塗れること」を使うのは結論を仮定する循環になる。健全性は**色を使わず
構造だけ**で定義する:

```
構造条件を満たす  ⟹  格子グラフに再構成できる  ⟹  平面地図
        〔健全性の定義はここまで〕                〔4色可は四色定理で従う結論〕
```

ツール: [squaretable.py](../fourcolor/squaretable.py) の `is_grid_realizable()`。
表から格子を再構成し（列ブロック=格子の行・行ブロック=格子の列・各ブロック1マス）、
**各列/行が単色・各国が辺連結・国境が表の差1と一致**を確認する。色は一切使わない。
成功すれば「格子由来 ⟹ 平面地図」、失敗すれば**どこで詰まるか**が分かる
（`python experiments/exp7_square_reconstruct.py`）:

- サンプル01（K4）・02（無色）… 格子に再構成できる（平面地図）✓
- exp6 の K5 … 再構成**できない**。理由「ある国（同値類）が辺連結でない（4マスが分裂）」

> 注: 表が持つのは**仮色値**で国番号ではない。「国」は**等式の同値類**
> （同じ横/縦ラベルでつながる有色セル）として構成する。`is_grid_realizable`
> は仮色値でなく同値類で辺連結性・国境を判定する（`_country_classes`）。

### 弱い条件では不十分（実験6・確定）

四角版の分かりやすい必要条件（**各ブロック≤1セル**・区切りは十字・同行同列は同色）
だけを満たすのに4色で塗れない表（K5）を狙い撃ちで構築できた
（`python experiments/exp6_square_soundness.py`）。

- 5つの国を5本のデータ行に割り当て、データ行の間に区切り行を入れて隣接させない。
  K5 の10辺は「2列ガジェット」（区切り列で隔てる）で作る。各ブロックは
  (2列)×(1データ行) で最大1セル。
- 結果: 弱い条件を満たすが4色不能・5色必要。

→ **四角でも局所条件だけでは不十分**（平面性は局所では捕まえられない）。
反例では国 i が「1本のデータ行に飛び飛びに散る**非連結**な領域」になっている。

### 候補の5条件でも不十分（実験8）

検討中の「分かりやすい5条件」:

1. ノード有セル（有色/無色）で**横ラベルが同じなら同色**
2. 同じく**縦ラベルが同じなら同色**
3. **横ラベルの差が1**の2つのノード有セルは同色にならない
4. **縦ラベルの差が1**の2つのノード有セルは同色にならない
5. **各ブロックにちょうど1個**のノード有セルが存在する（全ブロックが1セル）

- exp6 の K5 は**条件5を破る**（空きブロックだらけ）ので、5条件の反例ではない。
- だが全ブロックを埋めた表を全列挙すると、**5条件すべてを満たすのに格子に
  再構成できない表が存在する**（`python experiments/exp8_user_conditions.py`、
  A=2×B=3 で 896 個）。最小の反例は再構成すると

  ```
    2 1 2     ← この行の左右の「2」が分裂（間は別の国）。辺連結でない
    1 1 1        （数字は仮色値。左右の2マスは同じ列なので条件1で
                  必ず同じ同値類＝同じ国になる）
  ```

  表では「同じ列（横ラベル）に2マス → 条件1で同色」なので、この2マスは
  仮色値によらず**必ず同じ同値類（国）**。だが格子に戻すと非連結になる。
  原因は、格子の1行に沿って横ラベルが 3,4,3 と**非単調**になること
  （＝同じ横ラベルのマスが連続セグメントでない。実地図ではありえない）。

→ **5条件はまだ不十分**。足りないのは「各国が辺連結（＝各行・各列で連続区間、
ラベルが単調）」という性質。これは条件1〜2（同ラベル→同色）が、同ラベルのマスが
連続セグメントであることまでは強制しないために漏れる。

### 強い条件の核心: 各国が「辺連結（ポリオミノ）」か

四角の利点はここ。**各国がもとの格子で辺連結な領域なら、その表は文字どおり
平面地図（ピクセル化した地図）になり、四色定理から必ず4色で塗れる＝健全**。
上の反例が壊れるのは、まさに国が辺連結でない（飛び地的）から。

つまり四角版では、健全性の核心が**「条件が各国の辺連結性を強制するか」**の
1点に集約される。三角版（平面性を三角階段で間接的に縛る）より、健全性を
言い切りやすい見込み。

### 連続性条件（条件6）を足すと十分か（実験9）

条件6として「**各横ラベル（列）の有色マスが占める行ブロックは連続。各縦ラベル
（行）の有色マスが占める列ブロックも連続**」を足す（＝同じラベルのマスが格子上で
連続セグメントになる、というローカル寄りの条件）。

全ブロックを 2×2 で埋めた表を全列挙して、5条件+条件6 を満たすのに再構成
できない表が残るか調べた（`python experiments/exp9_contiguity.py`）:

| サイズ | 5条件OK | +条件6 OK | 再構成不可(反例) |
|---|---|---|---|
| A=2,B=2 | 192 | 192 | 0 |
| A=2,B=3 | 1984 | 1024 | 0 |
| A=3,B=2 | 1984 | 1024 | 0 |
| A=3,B=3 | 39040 | 6592 | 0 |

→ 条件6 は実際にフィルタとして効いており（3×3 で 39040→6592）、**残った表は
すべて格子に再構成できた**。**6条件で十分（健全）の有力な証拠**（小サイズの全列挙）。

### 6条件は必要条件か（実験10）

逆向き: **格子由来の表は必ず6条件を満たすか**。満たすなら「表の健全性 = 6条件」と
定義してよく、再構成のための追加条件を別途追わなくても、前向きの戦略
（6条件を満たす表をすべて4色で塗れれば、どんな地図も塗れる）が成り立つ。

各条件は格子の構造から従う（1,2=同セグメント同国 / 3,4=隣は国境で別国 /
5=ブロック=1マス / 6=セグメントは連続）。サンプル（無色入りの02 含む）と
ランダム連結格子地図3000個で機械確認した（`python experiments/exp10_necessity.py`）:
すべて6条件を満たし再構成可能。→ **6条件は必要条件**（強い証拠）。

これで「表の健全性 = 6条件」と置ける。ただし**注意**: 「6条件 ⟹ 4色可」を示すのは、
6条件がほぼ格子由来（平面地図）と一致する以上、**四色定理そのものと同じ難しさ**。
表現は問題を**正規化・観察しやすく**するが、易しくはしない（docs/01 以来の教訓）。
なお健全性の十分性（6条件 ⟹ 再構成可）は、前向き戦略では必須でないが、
彩色が詰まったとき「本物の地図か・条件不足の幻か」を切り分ける**診断**に役立つ。

### 残る課題

- 条件6 を入れても「**一般のサイズで必ず再構成成功**」の**証明**はまだ
  （全列挙は小サイズの証拠）。`is_grid_realizable()` の辺連結性チェックは BFS
  ＝大域的だが、条件6 はローカル寄り。「条件6 ⟹ 各国が辺連結」を組合せ的に
  証明できれば、健全性が確定する。
- 無色ノードを含む場合の条件6 の扱い（exp9 は有色のみで実験）。

## 失うもの・新たなデメリット（docs/09 §6 周辺の議論より）

- **次数が4**（三角は3）。局所更新には bounded で問題ないが、次数3の旨みは無い。
- **1点に集まれる国が4まで**（格子点に4マス。三角は6）。5カ国以上の点は無色
  ノードが必要になり、busy な地図で無色ノードが増える。
- **対角連結の厄介さ**（市松模様の点で同一国が斜めに分断される古典問題）。
- **Tait の3辺彩色との接続を失う**（[docs/08](08_実験2A_方向とビット分解.md) は
  3方向・cubic 固有）。
- 三角版の構造／健全性の議論（docs/09 §5,6、exp3〜5）は組み直し。
  四角は1セルブロックなので「格子＋併合は平面か」をより裸の形で観察できる利点もある。
